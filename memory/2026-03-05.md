# 2026-03-05

## Opus Effort 深度调查

### 问题发现
- **验证 Anthropic payload** — 使用 `OPENCLAW_ANTHROPIC_PAYLOAD_LOG` 检查实际请求
- **配置不生效** — 发现 `custom-v3-codesome-cn/claude-opus-4.6` 配置的 `effort=max` 实际发送的是 `effort=high`
- **根本原因** — OpenClaw 的 `mapThinkingLevel` 将 `adaptive` 映射为 `medium`，且 `supportsXHighThinking` 白名单不包含 anthropic/custom 提供商

### 解决方案
- **本地代理** — 创建 `scripts/codesome-effort-proxy.js`，强制重写 Opus → `effort=max`，Sonnet → `effort=high`
- **systemd 服务** — 通过 `codesome-effort-proxy.service` 管理代理进程
- **全局应用** — 更新所有 agent 的 `models.json`，将 `custom-v3-codesome-cn` 的 `baseUrl` 指向 `http://127.0.0.1:19002`
- **验证生效** — 确认代理成功重写 payload，Opus 请求现在使用 `effort=max`

### 文档输出
- **博客文章** — 发布了 `opus-effort-investigation.md` 到 qiuyuanblog
- **技术细节** — 记录了 OpenClaw 的 effort/thinking 映射机制和社区 workaround

### 技术洞察
1. OpenClaw `mapThinkingLevel` 在 agent session 前就映射 adaptive→medium
2. `supportsXHighThinking` 白名单排除了 anthropic/custom 提供商
3. `createStreamFnWithExtraParams` 忽略 effort/thinking 参数（只处理 temperature/maxTokens/transport/cache）
4. 上游 PR #27812 仍在开发中，当前需要本地 proxy/patch 作为 workaround
